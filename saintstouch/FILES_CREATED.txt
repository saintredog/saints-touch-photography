WHAT EACH FILE DOES:

CONFIG:
  astro.config.mjs
    → Tells Astro how to route localized versions
    → Sets English as default locale
    → Defines supported languages

TRANSLATION SYSTEM:
  src/i18n/utils.ts
    → t(locale, key) function for getting translations
    → Type-safe locale handling
    → Supports string interpolation

  src/i18n/locales/*.json
    → Actual translation strings for each language
    → Organized by section (nav, hero, portfolio, etc.)
    → 1,000+ total UI strings

COMPONENTS:
  LanguageSwitcher.astro
    → Dropdown to select language
    → Shows in header navigation
    → Links to same page in different language

  BaseLayoutI18n.astro
    → HTML wrapper with proper lang attribute
    → SEO hreflang tags
    → Integrates language switcher

PAGES:
  index.astro, es/index.astro, fr/index.astro, pt-br/index.astro, hi/index.astro
    → Localized versions of homepage
    → Each uses t(locale, key) for translations
    → Each includes language switcher

TESTING THE SETUP:

1. npm run dev
2. Visit http://localhost:3000/
3. Look for language switcher in header (top-right area)
4. Click it and select different languages
5. Page should translate to selected language
6. URL should change to /es/, /fr/, /pt-br/, or /hi/
7. Verify all sections translate properly

════════════════════════════════════════════════════════════════════════════
DEPLOYMENT:

1. npm run build
2. All localized versions output to dist/
3. Deploy dist/ to hosting provider
4. All routes will be available:
   - / (English)
   - /es/ (Spanish)
   - /fr/ (French)
   - /pt-br/ (Portuguese)
   - /hi/ (Hindi)

════════════════════════════════════════════════════════════════════════════
MAINTENANCE:

To add a new translation:
1. Add to all 5 JSON files in src/i18n/locales/
2. Use in template with t(locale, 'key')
3. Rebuild with npm run build

To add a new language:
1. Add to astro.config.mjs locales array
2. Create new .json file in locales/
3. Create locale-specific page folders
4. Follow same pattern as existing languages
